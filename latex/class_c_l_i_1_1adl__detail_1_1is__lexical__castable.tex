\subsubsection*{template$<$typename T, typename S = std::\+string$>$\newline
class CLI::\+adl\+\_\+detail::\+is\+\_\+lexical\+\_\+castable$<$ T, S $>$}
Check for existence of user-\/supplied lexical\+\_\+cast.

This struct has to be in a separate namespace so that it doesn\textquotesingle{}t see our lexical\+\_\+cast overloads in \doxylink{namespace_c_l_i_1_1detail}{CLI::\+detail}. Standard says it shouldn\textquotesingle{}t see them if it\textquotesingle{}s defined before the corresponding lexical\+\_\+cast declarations, but this requires a working implementation of two-\/phase lookup, and not all compilers can boast that (msvc, ahem).
